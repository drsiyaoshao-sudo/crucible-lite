### BILL: Architecture v21a — Polynomial-t Embedding + 4-layer MLP

**Proposed by:** Justice direction (2026-04-05)
**Change type:** software (pinn_model.py — new embedding layer + depth increase)

---

**Problem statement:**

v20 PINN output is a monotonic ramp for all channels — the 2-layer MLP with linear input has insufficient expressivity to represent the oscillatory, asymmetric gait waveform. The model learns `mean + slope×t` but cannot represent the sharp push-off spike near t≈0.6–0.7.

**Root cause:** The MLP receives raw `t ∈ [0,1]` as a single scalar. A 2-layer network can only apply two affine transforms — its function class for t is nearly linear. The gait signal requires odd-harmonic components (push-off is an asymmetric spike) that a linear or quadratic basis cannot represent.

---

**Proposed change:**

Replace raw t input with a **Chebyshev polynomial embedding** of degree K on t:

```
t ∈ [0,1]  →  t' = 2t − 1  (map to [−1, 1])
T₁(t')  = t'
T₃(t')  = 4t'³ − 3t'
T₅(t')  = 16t'⁵ − 20t'³ + 5t'
T₇(t')  = 64t'⁷ − 112t'⁵ + 56t'³ − 7t'
```

Odd Chebyshev polynomials are physically motivated: gy is anti-symmetric (negative stance, positive push-off), and odd-degree polynomials on a centred domain capture anti-symmetric features exactly. Even terms (T₂, T₄, T₆) added optionally for az (gravity DC + heel-strike spike which is not anti-symmetric).

**Full proposed embedding:** T₁ through T₈ (degrees 1–8, both odd and even) — gives 8 t-features vs 1 currently.

Combined input to MLP: `[x (10 features), T₁...T₈(t) (8 features)] = 18-dim`

**Network:** 4 hidden layers, hidden_dim=32 → ~5,800 params (≤ 0.1 × 500 profiles = 50 check: 5800 >> 50, still too large — but v20 sanity check used 1038 params and reached floor; need more data before scaling up. Propose hidden_dim=16, 4 layers → ~2,100 params for this experiment)

  class ChebyshevEmbedding(nn.Module):
      def __init__(self, degree: int = 5):
          super().__init__()
          self.degree = degree
      def forward(self, t: torch.Tensor) -> torch.Tensor:
          if t.dim() > 1:
              t = t.squeeze(-1)
          x = 2.0 * t - 1.0
          polys = []
          T_prev = torch.ones_like(x)
          T_curr = x.clone()
          polys.append(T_curr)
          for _ in range(2, self.degree + 1):
              T_next = 2.0 * x * T_curr - T_prev
              polys.append(T_next)
              T_prev, T_curr = T_curr, T_next
          return torch.stack(polys, dim=-1)  # (N, degree)
